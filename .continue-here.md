<decisions_made>

- All demos use handwritten data-demo-code="fail|pass" blocks (not live DOM serialisation) — more reliable, easier to read
- Interactive demos: fields start neutral, errors only shown after Submit click, oninput resets error state
- Combobox pattern follows ARIA 1.2: role="combobox" on input, role="listbox" on ul, role="option" on li, arrow key nav, Enter selects, Escape closes, Tab closes naturally
- Max 3 results shown in autocomplete dropdowns
- Singular/plural computed dynamically (never use "(s)" pattern)
- Screen reader announcements must avoid visual property references — describe functional impact only

</decisions_made>
