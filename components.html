How to use: Open in browser, toggle dark mode, tab through interactive elements, verify focus outlines visible, test arrow keys on combobox/tabs. Scan with axe: npx http-server . -p 8080 & then npx @axe-core/cli http://localhost:8080/components.html --tags wcag2aa

1. Demo Toggle Button

Binary fail/pass toggle used in all 45 checkpoint demos. Uses aria-pressed to communicate state to screen readers. Label text changes via hidden spans.

Interactive Example

Variants

Fail State (default)

Pass State

Code

<button class="demo-toggle" type="button" aria-pressed="false">
  <span class="demo-toggle__label--fail">Show passing example</span>
  <span class="demo-toggle__label--pass" hidden>Show failing example</span>
</button>

Keyboard Navigation

  • Tab Focus button
  • Space Toggle fail/pass state, announce via live region
  • Enter Toggle fail/pass state, announce via live region

Accessibility

Uses aria-pressed="true|false" to announce state to screen readers
Label text changes via hidden spans (no aria-label needed)
Works with demo-toggle.js live region announcements (data-announce-fail, data-announce-pass)

12. Theme Toggle Button

Button that switches page theme between light and dark modes. Uses aria-pressed and toggles data-theme attribute on <html>.

Interactive Example

Current theme: Light

Click to toggle light/dark mode (affects entire page)

Code

<button class="theme-toggle" aria-pressed="false">Dark mode</button>

<script>
button.addEventListener('click', () => {
  const isDark = html.getAttribute('data-theme') === 'dark';
  html.setAttribute('data-theme', isDark ? '' : 'dark');
  button.setAttribute('aria-pressed', !isDark);
  localStorage.setItem('theme', isDark ? 'light' : 'dark');
});
</script>

Keyboard Navigation

  • Tab Focus button
  • Space/Enter Toggle theme, update aria-pressed and localStorage

Accessibility

Uses aria-pressed to announce current state
Theme preference persisted in localStorage
Respects system prefers-color-scheme on first load
