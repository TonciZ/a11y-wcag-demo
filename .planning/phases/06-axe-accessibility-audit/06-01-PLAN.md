Create Node.js script that:
1. Enumerates all 44 checkpoint HTML files from checkpoints/ directory
2. Starts http-server on port 8765 (ephemeral or fixed)
3. For each checkpoint, invokes @axe-core/cli with standard: wcag2aa (AXE-01 requirement)
4. For each checkpoint, invokes pa11y to catch different violation classes
5. Outputs JSON results to .planning/audits/light/{checkpoint-name}.json (axe) and {checkpoint-name}-pa11y.json (pa11y)
6. Kills http-server after all scans complete

Script structure (pseudocode):
- Import: fs, path, child_process (spawn), axe, pa11y
- Enumerate: fs.readdirSync('checkpoints').filter(f => f.endsWith('.html'))
- Enumerate produces: ['1-1-1-non-text-content.html', '1-2-1-audio-only-...html', ...]
- For each file:
  - Build URL: http://localhost:8765/checkpoints/{filename}
  - Run axe: await axe.run(url, {standard: 'wcag2aa', resultTypes: ['violations', 'passes', 'incomplete']})
  - Run pa11y: await pa11y(url)
  - Write axe JSON: fs.writeFileSync(`.planning/audits/light/{name}.json`, JSON)
  - Write pa11y JSON: fs.writeFileSync(`.planning/audits/light/{name}-pa11y.json`, JSON)
- Gracefully handle server startup delay (setTimeout 1000ms)
- Catch and log errors per page (don't halt on single failure; continue scanning)
- Log progress: "Scanned {count}/44..." messages to stdout
- Gracefully shutdown: server.kill() when done

Expected output: .planning/audits/light/ directory with 88 JSON files (44 axe + 44 pa11y).

From RESEARCH.md — don't hand-roll JSON parsing; use Node.js APIs directly. axe-core and pa11y output schemas are stable.
