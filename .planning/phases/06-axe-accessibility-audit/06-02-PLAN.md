## Triage Classification Logic (from CONTEXT.md + RESEARCH.md)

**Intentional violation:** Any violation where `targetElement` is inside a `[data-mode="fail"]` container.
- These demonstrate accessibility failures (teaching examples)
- Do NOT fix; document in AUDIT-TRIAGE.md under "Intentional Violations"

**Real issue (fix immediately):** Violations in:
- Site chrome (header, footer, nav)
- Page structure (body, main, article)
- Elements outside any demo container
- Do NOT include violations in [data-mode="pass"] — those are deferred to Phase 7 UAT

**Real issue (deferred to Phase 7):** Violations in `[data-mode="pass"]` containers.
- These are broken passing examples (not teaching correctly)
- Document in AUDIT-TRIAGE.md under "Real Issues" with "DEFERRED" marker
- Fix only if Phase 7 UAT requires; out of scope for Phase 6
